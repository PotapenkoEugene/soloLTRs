## Documentation for Reproducibility

All methods, tools, and references discovered during research must be saved as separate markdown files in the knowledge base (`~/Orthidian/knowledge/soloLTRs/`) for full reproducibility and scientific paper writing. This includes:
- Tool surveys and comparisons (inputs, outputs, capabilities, limitations)
- Published methods and their citations (DOI, authors, year, journal)
- Ground truth datasets and where to download them
- Validation strategies and benchmark designs
- Key findings from literature review

Each file should be self-contained with full citations (DOI) so the information can be traced back to the source when writing the paper.
